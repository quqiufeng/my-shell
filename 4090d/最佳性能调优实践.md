## 二、模型方案对比

### 方案一：llama.cpp (Q4)

**配置**:
- 模型: Qwen2.5-Coder-32B-Instruct-Q4_K_M
- 文件: `/opt/gguf/qwen2.5-coder-32b-instruct-q4_k_m.gguf` (19GB)
- 量化: Q4_K_M (4bit)

**启动参数**:
```bash
llama-server \
  -m /opt/gguf/qwen2.5-coder-32b-instruct-q4_k_m.gguf \
  --host 0.0.0.0 \
  --port 11434 \
  --n-gpu-layers 80 \
  --ctx-size 65536 \
  --batch-size 1024 \
  --ubatch-size 512 \
  --flash-attn on \
  --cache-type-k q4_0 \
  --cache-type-v q4_0 \
  --threads 14 \
  --no-mmap \
  --mlock
```

**性能**: 42 tokens/s

**资源占用**:
- 显存: 23.8GB / 24GB
- 上下文: 64K

### 方案二：ExLlamaV2 + 投机采样 (推荐)

**配置**:
- 主模型: `/opt/gguf/exl2_4_0` (EXL2 4bit)
- 草稿模型: `/opt/gguf/Qwen2.5-Coder-0.5B-exl2`
- 投机Token数: 6
- 最大上下文: 32K

**启动命令**:
```bash
nohup python3 run_qwen2.5-coder-32b_exl2.py > /opt/exl2_api.log 2>&1 &
```

**核心参数**:
```python
config.no_flash_attn = False   # 启用FlashAttention
config.no_sdpa = False          # 强制用FlashAttention
cache = ExLlamaV2Cache_Q4()    # Q4 KV Cache (最快)
NUM_SPECULATIVE_TOKENS = 6      # 投机采样数
```

**性能**: 164 tokens/s

**资源占用**:
- 显存: 21.4GB / 24GB
- 上下文: 32K

## 五、优化建议

### 1. 显存优化

- `--no-mmap --mlock`: 减少页面交换，提升性能 ~20%
- GPU层数: 根据显存调整 (80-99)

### 2. 上下文配置

**Qwen2.5-Coder-32B 映射关系**:
| -c 参数 | 实际上下文 | 显存需求 |
|---------|-----------|----------|
| 32768 | 32K | 4.5GB |
| 65536 | 64K | 4.6GB (当前) |
| 131072 | 128K | 9GB (OOM) |

### 3. 投机采样

- 草稿模型: 0.5B 足够
- 投机数: 6 为最优
- 接受率: 取决于任务复杂度

### 4. mlock 限制

如遇警告 `failed to mlock`，可调整:
```bash
ulimit -l unlimited  # 需root权限
```

## 七、场景推荐

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 代码补全/生成 | ExLlamaV2 | 164 tok/s, 速度快 |

## 八、注意事项

1. FlashAttention 必须安装
2. 显存剩余 < 1GB 时可能出现 OOM
3. 投机采样首次加载较慢
4. 推理时 GPU 利用率会波动
